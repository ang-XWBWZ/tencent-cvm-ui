# 地区选择功能设计文档

## 概述
改进前端地区选择功能，将地区选择集成到头部栏，使用弹窗选择，支持持久化存储和服务器数量显示。

## 需求分析
1. **头部栏集成**：在头部栏添加"更改地区"按钮，格式为"地区-区域【更改地区】"
2. **弹窗选择**：复用现有的地区选择逻辑，增加搜索功能
3. **持久化存储**：存储地区列表和用户选择，支持localStorage和cookie
4. **全局共享**：地区选择在整个应用中全局共享
5. **服务器数量显示**：在服务器管理界面显示地区服务器数量

## 设计决策

### 1. 整体架构
- 在App.vue中添加全局地区状态（selectedRegion）
- 头部栏显示当前地区和【更改地区】按钮
- 点击按钮打开地区选择弹窗
- 地区状态通过props传递给子组件
- InstanceManagement中显示地区服务器数量

### 2. 头部栏修改
- 在App.vue的header-actions区域添加地区显示
- 显示格式：'上海【更改地区】'（当前地区+按钮）
- 点击【更改地区】打开地区选择弹窗

### 3. 弹窗设计（智能模式）
- **购买页面**：显示地区和区域两个下拉列表
- **其他页面**：只显示地区选择
- 弹窗包含：
  - 地区选择（el-select）
  - 区域选择（el-select，按需显示）
  - 搜索框（可搜索地区名称）
  - 刷新按钮（重新从API加载地区列表）
  - 确认/取消按钮
- 选择地区后自动加载该地区的区域列表
- 地区列表从持久存储加载，为空时从API加载

### 4. 持久化设计
- 存储完整的地区-区域映射
- 扩展cache.js，添加地区专用函数：
  - `saveRegionData(regions, selectedRegion, regionZoneMap)`
  - `loadRegionData()` 返回{regions, selectedRegion, regionZoneMap}
  - `clearRegionData()`
- 存储策略：优先localStorage，失败时降级到cookie
- 存储内容：
  - 地区列表（数组）
  - 用户选择的地区（字符串）
  - 地区-区域映射（对象）
  - 时间戳（用于过期检查）
- 地区列表过期时间：24小时（可配置）
- 搜索功能从持久化数据中搜索，避免API调用

### 5. 服务器数量显示
- **两种方式都提供**：
  1. 在地区选择控件中显示数量（下拉框选项后显示"上海 (15台)"）
  2. 独立统计面板（在服务器管理页面添加地区统计面板）
- 需要API支持：获取每个地区的服务器数量统计
- 实现考虑：批量查询、定期刷新、性能优化

### 6. 集成设计
- **统一管理**：移除InstanceManagement.vue中的地区选择控件，使用全局地区
- **页面集成**：
  - InstanceManagement：接收地区props，显示服务器数量，添加统计面板
  - 购买页面：地区选择与全局地区同步
- **文件修改**：
  1. App.vue：添加全局地区状态、头部栏按钮、弹窗组件
  2. InstanceManagement.vue：使用传入的地区，显示服务器数量
  3. cache.js：扩展持久化功能
  4. api.js：添加获取地区服务器数量的API
  5. 新增RegionDialog.vue（弹窗组件）

## 技术实现

### 组件结构
```
App.vue
├── 头部栏（地区显示 + 【更改地区】按钮）
├── RegionDialog.vue（地区选择弹窗）
├── 全局状态：selectedRegion, regions, regionZoneMap
└── 子组件：
    ├── InstanceManagement.vue（接收地区props，显示服务器数量）
    ├── BillingManagement.vue（接收地区props）
    └── 购买页面（地区选择与全局同步）
```

### 状态流
1. 应用启动时从持久存储加载地区数据和用户选择
2. 头部栏显示当前地区
3. 点击【更改地区】打开弹窗
4. 在弹窗中选择地区（和区域）
5. 确认后更新全局状态并保存到持久存储
6. 所有监听地区变化的组件自动更新

### API扩展
1. 获取地区服务器数量：`GET /api/regions/instance-counts`
2. 批量获取地区信息：`GET /api/regions/details`（包含区域列表）

## 风险与缓解

### 风险1：地区数据量大
- 缓解：分页加载、懒加载、压缩存储

### 风险2：cookie大小限制
- 缓解：优先localStorage，cookie只存储关键数据

### 风险3：API性能
- 缓解：缓存、批量查询、定期刷新策略

### 风险4：浏览器兼容性
- 缓解：特性检测、降级方案

## 验收标准
1. 头部栏正确显示当前地区和【更改地区】按钮
2. 点击按钮弹出地区选择弹窗
3. 弹窗根据当前页面智能显示地区/区域选择
4. 地区选择后自动保存，刷新页面后恢复
5. InstanceManagement正确显示地区服务器数量
6. 搜索功能正常工作
7. 刷新按钮能重新加载地区数据
