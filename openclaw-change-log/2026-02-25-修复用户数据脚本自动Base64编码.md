# 2026-02-25：修复用户数据脚本自动 Base64 编码

## 背景

在创建 CVM 时，前端输入的是“纯文本用户数据脚本”，但后端直接将原文传给腾讯云 `RunInstancesRequest.UserData`，导致创建实例时报错。

## 变更内容

- 文件：`src/main/java/com/example/tengxuncvm/service/CvmInstanceCreateService.java`
- 调整：在设置 `runRequest.setUserData(...)` 前，先将脚本文本按 UTF-8 编码后进行 Base64 转换。

核心逻辑：

```java
String userDataBase64 = Base64.getEncoder()
        .encodeToString(request.getUserData().getBytes(StandardCharsets.UTF_8));
runRequest.setUserData(userDataBase64);
```

## 结果

- 前端可继续输入正常脚本（例如 `#!/bin/bash` 开头的 shell 内容）；
- 后端自动转换为腾讯云 API 需要的 Base64 字符串后再提交。
